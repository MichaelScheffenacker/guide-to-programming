# The Idea Machine (Ideenmaschine)

Putting Ideas into the machine, like gears into a clockwork, to make this Ideas to evolve and unfold with every stroke.  This is the essence of programming. 

Programming is a truly transcendent task.  The Ideas get executed in a world that is separated from ours.  And this ideas can evolve into fantastic constructs, into worlds of their own.



When I read my first book about mathematics, I was introduced to the idea of sets.  And this book claimed that anything could be part of a set, even a chair and a table.  When I heard my first mathematics lecture, the professor told us, that anything could be in a set with one restriction: an object can only be an element of a set, if it is possible to decide if it is in the set or not.  that sounds like an odd statement.  Can we decide if a chair is in a set or not.  Since we can not define what a chair is, and we can not know where it begins or it ends, it seems to be obvious that we can not clearly decide and therefore it can not be in a set.  But the idea of a chair can be in a set.  And as well can the idea of a chair be in a program.  And if we formed this idea properly, and combined it with other powerful ideas and eke out all inconsistencies and dead ends it will transform into something useful possibly even beautiful, stroke by stoke of the clock of our powerful machine.



A digital computer executes the program stroke by stroke of its internal clock.  The program evolves discretely from one state to the next state.  A digital computer consists of this clock; a memory that consists of a long string of Zeros and Ones; a central processing unit with a small set of instructions that operate on this memory; and a way to communicate with the rest of the world.
